# 自主执行系统 — 客户端手工测试方案

## 环境准备

### 前置条件
- Node >= 22, pnpm >= 10, Rust >= 1.94
- LLM API Key 已配置（通过 Setup 向导或 `~/.healthclaw/evo_claw.json`）
- 至少创建了一个 Agent

### 启动应用
```bash
cd /Users/mac/src/github/jone_qian/EvoClaw
pnpm dev        # EvoClaw 品牌
# 或
pnpm dev:healthclaw  # HealthClaw 品牌
```

### 开启日志监控（另开终端）
```bash
# HealthClaw 品牌
tail -f ~/.healthclaw/logs/core.log | grep -E "heartbeat|cron|system-event|BOOT"

# EvoClaw 品牌
tail -f ~/.evoclaw/logs/core.log | grep -E "heartbeat|cron|system-event|BOOT"
```

## 测试 1: HeartbeatManager 启动验证

**目的**: 确认 sidecar 启动时 HeartbeatManager 正确初始化

**步骤**:
1. 启动应用 `pnpm dev`
2. 观察终端日志

**预期日志**:
```
[INFO ] [heartbeat-manager] HeartbeatManager 已为 N 个 Agent 初始化
```

**验证点**:
- [ ] 日志出现 `HeartbeatManager 已为 X 个 Agent 初始化`
- [ ] X = 你已创建的 Agent 数量

---

## 测试 2: 心跳启用与配置

**目的**: 通过 UI 启用心跳并验证配置生效

**步骤**:
1. 进入 **专家中心** → 点击 Agent 卡片的 **「编辑」** 按钮
2. 右侧面板打开 → 点击 **「自动化」** tab
3. 打开 **💓 心跳巡检** 开关
4. 设置参数:
   - 执行间隔: **1** 分钟（方便测试）
   - 活跃时段: **00:00 - 23:59**（全天）
   - 告警投递: **不投递**
5. 观察终端日志

**预期日志**:
```
[INFO ] [heartbeat-manager] agent xxx heartbeat 已启动 (间隔=1m)
```

**验证点**:
- [ ] 开关打开后日志显示 heartbeat 启动
- [ ] 间隔和活跃时段参数正确

---

## 测试 3: 心跳执行 — HEARTBEAT_OK 零污染

**目的**: 空清单时心跳返回 HEARTBEAT_OK，不产生任何可见对话记录

**步骤**:
1. 确保心跳已启用（测试 2）
2. 进入 **「设置」** tab → 找到 **HEARTBEAT.md** → 确认内容为默认模板（无实际检查项）
3. 等待 1 分钟（心跳间隔）
4. 观察终端日志
5. 回到 **对话页面** → 检查对话列表

**预期**:
```
[DEBUG] [heartbeat] agent xxx heartbeat 完成
```

**验证点**:
- [ ] 日志显示心跳执行
- [ ] 对话列表中 **没有** 出现新的 heartbeat 对话（零污染）

---

## 测试 4: 心跳执行 — 有实际工作

**目的**: 心跳检测到需要做的事时，产生对话记录

**步骤**（二选一）:

**方式 A — 对话中让 Agent 自己改**:
1. 在对话中对 Agent 说：`把 HEARTBEAT.md 改成：每次心跳报告当前时间，并给我一句鼓励的话`
2. Agent 会自动用工具编辑 HEARTBEAT.md

**方式 B — UI 直接编辑**:
1. 进入 **「设置」** tab → 编辑 **HEARTBEAT.md**，改为:
   ```markdown
   # 定时检查

   ## 清单
   - 报告当前时间和日期
   - 给用户一句鼓励的话
   ```

两种方式效果一样，然后:
2. 等待下一次心跳（约 1 分钟）
3. 观察终端日志
4. 回到对话页面检查

**预期**:
- Agent 执行了清单中的任务（报告时间 + 鼓励语）
- 对话列表中出现 heartbeat 相关的对话记录

**验证点**:
- [ ] 日志显示心跳执行
- [ ] 对话列表出现新记录
- [ ] 记录内容包含时间和鼓励语

## 测试 5: 心跳活跃时段过滤

**目的**: 非活跃时段心跳应被跳过

**步骤**:
1. 修改活跃时段为一个当前时间之外的窗口，例如 **03:00 - 04:00**
2. 等待 1 分钟
3. 观察日志

**预期**: 无心跳执行日志（被跳过）

**步骤（恢复）**:
4. 改回 **00:00 - 23:59**

**验证点**:
- [ ] 非活跃时段无心跳执行
- [ ] 恢复后心跳正常执行

## 测试 7: 定时任务（Cron）— 独立执行

**目的**: 创建 prompt 类型的定时任务并验证执行

**步骤**:
1. **「自动化」** tab → 定时任务区域 → 点击 **「+ 新建」**
2. 填写:
   - 任务名称: `测试问候`
   - Cron 表达式: `* * * * *`（每分钟）
   - 类型: **独立执行**
   - 执行内容: `请用一句话问候用户`
3. 点击 **创建**
4. 等待 1 分钟
5. 观察日志

**预期日志**:
```
[DEBUG] [cron] cron job 测试问候 执行完成
```

**验证点**:
- [ ] 任务出现在列表中，显示下次执行时间
- [ ] 1 分钟后日志显示执行
- [ ] 测试完成后 **删除该任务**（避免持续消耗 token）

## 测试 8: 定时任务（Cron）— 事件注入模式

**目的**: event 类型的定时任务注入系统事件到主会话

**步骤**:
1. **「自动化」** tab → **「+ 新建」** 定时任务
2. 填写:
   - 任务名称: `日程提醒`
   - Cron 表达式: `* * * * *`（每分钟）
   - 类型: **事件注入**
   - 执行内容: `提醒：10 分钟后有一个会议`
3. 创建后等待 1 分钟
4. 然后在 **对话页面** 向该 Agent 发一条普通消息，如 `你好`

**预期**:
- Agent 的回复应该提到会议提醒（因为事件被注入了 prompt）

**验证点**:
- [ ] 日志显示 `cron job 日程提醒 → system event 已注入`
- [ ] 发送消息后 Agent 回复中包含会议提醒的内容
- [ ] 测试完成后 **删除该任务**

## 测试 9: Standing Orders

**目的**: 验证 AGENTS.md 中的 Standing Orders 在心跳时被执行

**步骤**:
1. 确保心跳已启用（间隔 1 分钟）
2. **「设置」** tab → 编辑 **AGENTS.md** → 找到 `## Standing Orders` section
3. 在注释下方添加:
   ```markdown
   ### Program: 时间播报
   - **Scope**: 告诉用户当前的精确时间
   - **Trigger**: heartbeat
   - **Approval**: 无
   - **Escalation**: 无
   ```
4. 同时确保 HEARTBEAT.md 清单为空（或只有注释），这样 Standing Orders 是唯一的工作来源
5. 等待下一次心跳

**预期**:
- Agent 执行了"时间播报"程序，回复中包含当前时间
- 由于有实际输出，对话列表出现新记录

**验证点**:
- [ ] 心跳执行时 Agent 报告了时间
- [ ] 对话列表出现 heartbeat 记录

## 测试 10: BOOT.md 启动执行

**目的**: sidecar 启动时自动执行 BOOT.md

**步骤**:
1. **「设置」** tab → 编辑 **BOOT.md**（如果 UI 中未显示，通过文件系统直接编辑）
   ```bash
   # 找到工作区路径
   ls ~/.healthclaw/agents/*/workspace/BOOT.md
   # 编辑（替换实际路径）
   echo '# 启动任务
   - 向用户问好，告诉他你已经上线了' > ~/.healthclaw/agents/<agent-id>/workspace/BOOT.md
   ```
2. **重启应用**（Ctrl+C → `pnpm dev`）
3. 观察日志

**预期日志**:
```
[INFO ] [server] Agent xxx BOOT.md 已触发执行
```

**验证点**:
- [ ] 启动日志中出现 BOOT.md 执行记录
- [ ] 对话列表可能出现 boot 会话记录

3. 等待心跳触发

**预期**:
- 微信收到 Agent 的心跳输出消息

**验证点**:
- [ ] 微信收到消息
- [ ] 消息内容与心跳输出一致

## 测试完成清理

测试结束后恢复默认状态：

1. **关闭心跳** → 自动化 tab → 关闭开关
2. **删除测试 Cron 任务** → 点各任务的删除按钮
3. **恢复 HEARTBEAT.md** → 设置 tab → 恢复为默认模板
4. **恢复 AGENTS.md** → 删除测试用的 Standing Orders Program
5. **清空 BOOT.md** → 删除测试内容或恢复为注释

## 问题排查

| 症状 | 可能原因 | 排查方法 |
|------|---------|---------|
| 心跳不执行 | 未配置 LLM provider | 检查 Setup 向导是否完成 |
| 心跳不执行 | 活跃时段限制 | 改为 00:00-23:59 |
| 心跳不执行 | 间隔门控（默认 5 分钟内不重复） | 等待超过 5 分钟再测 |
| 对话列表无记录 | HEARTBEAT_OK 零污染（正常行为） | 在 HEARTBEAT.md 写具体任务 |
| Cron 不执行 | 表达式错误 | 用 `* * * * *` 测试 |
| BOOT.md 不执行 | 内容为空/仅注释 | 写非注释内容 |
| 日志无输出 | 日志路径错误 | 检查品牌对应目录（.healthclaw/.evoclaw） |
