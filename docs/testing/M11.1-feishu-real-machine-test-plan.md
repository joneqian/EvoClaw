## Part 1 — 连接与认证（PR1 能力验证）

### T1.1 ✅/❌ 正确凭据能连上
**前置**: `pnpm dev` 启动，打开桌面 app
**操作**:
1. 前端 ChannelPage → 飞书 → 填 App ID + App Secret + Domain=feishu → 绑定测试 Agent → 点连接
**预期**:
- UI 显示"已连接"
- sidecar 日志：`[INFO] [feishu-adapter] 飞书 Channel 已连接 appId=cli_xxx bot=ou_xxx`
- `bot=ou_xxx` 不是 `未知`（表示 `/open-apis/bot/v3/info` 调通）
**失败排障**:
- `bot=未知` → 检查权限是否发布生效
- `连接失败: code=99991400` → rate limit，等 1 分钟
- `连接失败: 飞书配置不合法` → 凭据填错

### T1.2 ✅/❌ 错误凭据拒绝连接
**操作**: 故意把 App Secret 填错后连接
**预期**:
- UI 显示"错误：...（飞书 code=xxx）"
- sidecar 日志有 error，无 WS 长连接建立
- 状态保持 disconnected

### T1.3 ✅/❌ Keychain 冷启动链路
**操作**:
1. T1.1 成功后，退出桌面 app + 终止 sidecar
2. 重启桌面 app（触发 sidecar 自动恢复连接）
**预期**:
- 日志：`渠道 feishu 已自动恢复连接`
- UI 上飞书状态仍为已连接
**失败排障**: channel_state 表的 credentials 列是否存在明文（PR1 followup 加密前仍是明文）

### T1.4 ✅/❌ 租户域名切换（海外 Lark）
**仅海外租户**: 切 Domain = `lark` 重连
**预期**: 能连上 `open.larksuite.com` 而非 `open.feishu.cn`

## Part 5 — 审批卡片（PR3 + PR6）

### T5.1 ✅/❌ Agent 请求审批
**操作**:
让 Agent 执行一个需要审批的操作（比如让 AGENTS.md 里写规则"删除文件前必须审批"，然后让 Agent 删文件）
**预期**:
- Agent 调用 `feishu_request_approval` 工具
- 飞书端收到审批卡（orange 主题 + "批准"/"拒绝" 按钮）
- **验证**: 卡片标题与正文是 Agent 传的（不是"审批"/"已收到决定：approve" 默认值，PR3 修过的 HIGH）
- Agent 处于 await 状态

### T5.2 ✅/❌ 用户点"批准" → Agent 继续
**操作**: 测试用户点"批准"按钮
**预期**:
- 卡片立即变为 green 主题 + "已批准（操作者 ou_xxx）"
- Agent Promise resolve `{decision: 'approve', operatorOpenId: 'ou_xxx'}`
- Agent 继续执行后续操作

### T5.3 ✅/❌ 用户点"拒绝" → Agent 中止
**操作**: 重复 T5.1，这次点"拒绝"
**预期**:
- 卡片变为 red 主题 + "已拒绝（操作者 ou_xxx）"
- Agent 收到 `decision: 'deny'` 并中止操作

### T5.4 ✅/❌ TTL 超时自动转 timeout
**操作**: Agent 发起审批时 `ttlMs: 30_000`，等 30s 不点
**预期**:
- 卡片自动变为 grey 主题 + "已超时"
- Agent 收到 `decision: 'timeout'`

### T5.5 ✅/❌ 非授权用户点按钮被忽略
**仅限定 operator 场景**: Agent 发起时传 `operatorOpenId: 'ou_admin'`，让非 admin 用户点
**预期**: 日志 `卡片 action 被拒：operator_mismatch`，卡片状态不变

## Part 8 — 稳定性（PR5 retry + PR1 重连）

### T8.1 ✅/❌ 断网 30s 自动重连
**操作**:
1. 飞书连接成功后，**拔网线或关 WiFi** 30 秒
2. 重新连网
**预期**:
- 日志有重连尝试：`[ChannelManager] feishu 尝试重连 (1/10)...`
- SDK 内部自动重连 + EvoClaw `ChannelManager` 外层退避
- 重连成功：`飞书 Channel 已连接`
- 重连后 T2.1 单聊能正常收发

### T8.2 ✅/❌ 限流场景观察（难模拟，日志核对）
**操作**: 让 Agent 短时间内连发 20 条消息
**预期**:
- 大概率不会触限流（飞书 200 QPS）
- 如果出现 `code=99991400` → 日志看到 `[feishu-retry] [sendMessage] 第 1/3 次失败，xxxx ms 后重试`
- 退避延迟应含 jitter（不是整齐 1000/2000/4000，而是随机 50%-100%）

### T8.3 ✅/❌ 不可重试错误直接抛
**操作**: 让 Agent 传给 `feishu_send` 一个**不存在的 peerId** `ou_notexist`
**预期**:
- 飞书返回 230002 `invalid receive_id`
- 不重试（log 无 retry warn）
- 错误原样抛给 Agent

## 排障决策树

```
发现问题
├─ 完全收不到任何消息
│  → 90% 是飞书后台事件未勾选（见 0.3）
│  → 次高：bot open_id 未发现，群 @ 过滤拦截
│  → 见 T1.1 bot 字段
│
├─ 机器人能收但发不出去
│  → receive_id_type 错配 / peerId 被 scope 污染
│  → 看日志 receive_id 值 vs 飞书原始 chat_id
│
├─ 群 @ 机器人无响应
│  → bot.open_id 未识别（T1.1 验证 bot= 不是"未知"）
│  → 飞书 mention 数据结构变化（PR1 修的 union_id/user_id 都匹配应覆盖）
│
├─ 媒体下载失败
│  → 权限 im:resource 未加
│  → 机器人不在消息所在会话中
│  → 文件超过 100MB 上限
│
├─ 审批卡按钮无反应
│  → card.action.trigger 事件未勾选
│  → operator open_id 不匹配（T5.5 刻意场景）
│  → envelope 过期（默认 24h，但如果时钟不同步）
│
└─ 文档评论收不到
   → drive.notice.comment_add_v1 未勾选
   → drive:drive / drive:file.comment 权限未发布
   → 机器人不是文档的协作者
```
